# 0G

**Prize Pool:** $15,000

## About

0G is the intelligent Layer 1 for onchain AI, powered by the first decentralized AI operating system (dAIOS), and provides a complete infrastructure stack for building AI-powered applications onchain.

Developers can use 0G's four core components:

- 0G Storage
- 0G DA (infinitely scalable data availability layer)
- 0G Compute Network (decentralized AI inference and training)
- 0G Chain (EVM-compatible blockchain)

## Prize Tracks

### 🛠️ Best Agent Framework, Tooling & Core Extensions — $7,500

**Placements:**

- 🥇 1st place: $2,500
- 🥈 2nd place: $2,000
- 🥉 3rd place: $1,500
- 🏅 4th place: $1,000
- 🏅 5th place: $500

This track is strictly for framework-level work.

Build the best core extensions, improvements, forks, or entirely new open agent frameworks inspired by OpenClaw (or alternatives like ZeroClaw, NullClaw, etc.) and deployed on 0G. Focus on advancing how agents are created in 2026: architectures, developer tooling, and infrastructure primitives that other builders will use.

**Ideas to get started:**

- New OpenClaw modules for hierarchical planning, reflection loops, or multimodal reasoning that natively integrate 0G Compute's sealed inference (live models like `qwen3.6-plus` or `GLM-5-FP8`)
- A self-evolving agent framework that autonomously generates, tests, and integrates new skills/tools using persistent 0G Storage memory
- A modular "agent brain" library with easy swapping of memory layers (0G Storage KV/Log), LLM backends, or decision engines
- A no-code/low-code visual agent builder with one-click deployment to 0G Compute + Storage

**Qualification requirements:**

- Project name and short description
- Contract deployment addresses
- Public GitHub repository (`README` + setup instructions)
- Demo video and live demo link (video under 3 minutes)
- Explanation of which protocol features or SDKs were used
- Team member names and contact info (Telegram and X)

**You must also include:**

- At least one working example agent built using your framework/tooling (example code or link in `README`)
- An architecture diagram (optional but strongly recommended) showing integration with OpenClaw + 0G Storage/Compute
